La diferencia entre usar #include<> e #include”” se encuentra en el lugar en el que el preprocesador busca a los archivos, cuando ponemos <> el pre-porcesador busca en las carpetas pre-definidas por el compilador mientras que el uso de “” implica que el preprocesador buscará primero en la carpeta origen donde se encuentra el programa .c que está llamando y posteriormente a las rutas pre-definidas, normalmente se usa para incluir header files definidos por el programador. 
La palabra static previa a la definición de una función o una variable fuera de una función implica que las funciones o variables definidas tienen alcance solamente local, es decir solamente pueden ser accesibles desde el mismo archivo en el que se definieron lo que implica que son privadas en relación a otros archivos de C. 
Por otro lado si definimos una variable como static dentro del alcance de una función, entonces esta variable es accesible por las demás funciones dentro del mismo archivo por ejemplo supongamos que tenemos una función que incrementa el valor de una variable static int x en uno cada vez que es llamada y que tiene otra variable definida de forma usual int y, si en el main llamamos la función tres veces, el valor de x irá incrementando ya que los cambios se guardan a lo largo del file para las variables static dentro de las funciones mientras que el valor de y siempre será de 1.

2.- Investiga* sobre BLAS, CBLAS, LAPACK, ATLAS y las operaciones de nivel 1, nivel 2 y nivel 3 de BLAS y reporta sobre esta investigación que realizas. Es una investigación que contiene principalmente una descripción sobre los paquetes y niveles anteriores.
BLAS: Basic Linear Algebra Subprograms, es un conjunto de rutinas que proveen bloques de código sobre los cuales construir rutinas complejas para ejecutar operaciones de matrices y vectores, el nivel 1 de BLAS realiza operaciones por escalares y operaciones vector-vector, el nivel 2 realiza operaciones matriz-vector y por último el nivel 3 realiza operaciones del tipo matriz-matriz, estas subrutinas son portables, eficientes y estables y se utilizan en un gran número de paquetes de algebra como por ejemplo LAPACK. 
CBLAS es una interfaz estilo C a las rutinas de blas. 
LAPACK es una rutina escrita en FORTRAN 90 y permite resolver sistemas de ecuaciones lienales simultaneas, solucione spor mínimos cuadrados, y en general problemas complejos que involucran matrices (LU,QR,SVD ..etc), el objetivo original de LAPACK era el de en vectores de memoria compartida o en procesadores en paralelo.
